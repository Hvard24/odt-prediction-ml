# Machine Learning Prediction of Aqueous Odor Detection Thresholds

This repository accompanies the study:

**"Machine learning reveals molecular features associated with odor detection thresholds and identifies high-potency food odorants"**

It provides curated datasets and reproducible Python scripts for predicting aqueous odor detection thresholds (ODTs) of organic compounds using machine learning.

## Summary

* **Dataset:** 1003 compounds with curated aqueous ODTs
* **Endpoint:** aqueous odor detection threshold expressed as pT
* **Model:** XGBoost regression model
* **Validation:** internal train/test validation, 5-fold cross-validation, external validation, and prospective screening
* **External validation:** 177 compounds not included in model training
* **Additional analyses:** descriptor filtering, hyperparameter optimization, applicability-domain assessment, and prediction uncertainty analysis

## Data Description

### Table S1 – Curated Benchmark Dataset

`Table_S1_dataset.csv` contains the final curated aqueous ODT dataset used for model development.

This table includes compound identifiers, canonical SMILES, curated aqueous ODT values, pT values, and related information used to support dataset traceability and reproducibility.

For compounds with multiple reported aqueous ODT values, the final consensus threshold was calculated using geometric averaging after conversion to mg/L.

### Table S2 – External Validation Dataset

`Table_S2_external_validation.csv` contains compounds used for external validation.

These compounds were not included in model training and were used to evaluate the generalization ability of the trained model.

### Table S3 – Screening Candidates

`Table_S3_screening_candidates.csv` contains candidate compounds used for model-based screening and prospective prediction.

The trained model can be applied to this table using `predict_screening.py`.

### Table S4 – Descriptor Matrix

`Table_S4_descriptor_matrix.csv` contains molecular descriptors calculated from canonical SMILES strings using RDKit.

This file is generated by:

```bash
python code/descriptor_calculation.py
```

The descriptor matrix is used as the input feature table for model training and prediction.

### Table S5 – Applicability Domain and Prediction Uncertainty

`Table_S5_applicability_domain_uncertainty.csv` contains applicability-domain and prediction-uncertainty results.

This table includes information such as structural similarity, nearest-neighbor relationships, applicability-domain status, and uncertainty-related outputs for validation and screening compounds.

## Requirements

The main Python packages required are:

```bash
pip install pandas numpy scikit-learn xgboost joblib
```

RDKit is required for molecular descriptor calculation. Installation via conda is recommended:

```bash
conda install -c conda-forge rdkit
```

## Usage

### 1. Descriptor Calculation

Calculate RDKit molecular descriptors and generate `Table_S4_descriptor_matrix.csv`:

```bash
python code/descriptor_calculation.py
```

### 2. Model Training

Train the XGBoost regression model using the benchmark dataset and descriptor matrix:

```bash
python code/train_model.py
```

The trained model and related outputs will be saved in the `outputs/` folder, including:

```text
xgboost_best_model.pkl
imputer.pkl
final_selected_features.pkl
xgboost_metrics_report.json
```

Depending on the version of the script used, additional files such as cross-validation results, selected features, or Y-randomization results may also be generated.

### 3. Prediction for Screening Candidates

Apply the trained model to candidate compounds in `Table_S3_screening_candidates.csv`:

```bash
python code/predict_screening.py
```

The prediction results will be saved as:

```text
outputs/screening_predictions.csv
```

## Target Definition

The prediction target is defined as:

```text
pT = -log10(ODT × 10^-3)
```

where ODT is expressed in mg/L.

The inverse transformation is:

```text
ODT_mg/L = 10^(3 - pT)
```

## Model Development

The model-development workflow includes:

1. Molecular descriptor calculation using RDKit
2. Removal of descriptors with excessive missing values
3. Removal of zero-variance descriptors
4. Mean imputation for remaining missing descriptor values
5. Train/test split using a fixed random seed
6. Hyperparameter optimization using grid search
7. Final model training using XGBoost
8. Evaluation using R², RMSE, and MAE

## Reproducibility

All scripts are provided to reproduce descriptor calculation, model training, and candidate prediction.

A fixed random seed is used where applicable:

```text
random_state = 42
```

Minor differences in model performance may occur because of software versions, RDKit descriptor calculation, and machine-learning package updates.

## Notes

The SHAP-based and descriptor-based analyses should be interpreted as model-derived physicochemical associations rather than direct receptor-level mechanistic evidence.

The prospective screening results are intended to support model-guided prioritization of candidate high-potency odorants.
